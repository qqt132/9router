# OpenAI 批量注册使用说明

## 📋 功能概述

修改后的 `register.py` 支持批量串行注册 OpenAI 账号，每个账号注册完成后自动等待指定间隔再继续下一个。

## 🚀 使用方法

### 基础用法（使用默认参数）

```bash
python3 register.py
```

默认行为：
- 注册 5 个账号
- 每个账号间隔 3 秒
- 使用 tempmail 邮箱服务
- 结果保存到 `openai-accounts-YYYYMMDD-HHMMSS.json`

### 自定义参数

```bash
# 注册 10 个账号，间隔 5 秒
python3 register.py --count 10 --interval 5

# 使用代理
python3 register.py --count 5 --proxy http://127.0.0.1:7890

# 指定输出文件
python3 register.py --count 3 --output my-accounts.json

# 静默模式（只输出 JSON）
python3 register.py --count 5 --quiet
```

## 📊 参数说明

| 参数 | 说明 | 默认值 | 示例 |
|------|------|--------|------|
| `--count` | 注册账号数量 | 5 | `--count 10` |
| `--interval` | 账号间隔秒数 | 3 | `--interval 5` |
| `--email-service` | 邮箱服务 | tempmail | `--email-service tempmail` |
| `--proxy` | 代理地址 | 无 | `--proxy http://127.0.0.1:7890` |
| `--output` | 输出文件路径 | 自动生成 | `--output accounts.json` |
| `--quiet` | 静默模式 | 关闭 | `--quiet` |

## ⚠️ 注意事项

1. **串行执行**：账号按顺序逐个注册，不是并行
2. **错误处理**：单个账号失败不影响后续账号
3. **间隔时间**：建议间隔至少 3 秒，避免触发频率限制
4. **代理使用**：如果 IP 地区不支持，需要使用代理
5. **日志保存**：每个账号的详细日志都保存在结果文件中

## 🔧 故障排查

### 问题：所有账号都失败

**可能原因**：
- IP 地区不支持（需要使用代理）
- 网络连接问题
- 邮箱服务不可用

**解决方案**：
```bash
# 使用代理重试
python3 register.py --proxy http://127.0.0.1:7890
```

### 问题：部分账号失败

**可能原因**：
- 验证码接收超时
- 临时网络波动
- OpenAI 服务限流

**解决方案**：
- 增加间隔时间：`--interval 5`
- 减少批量数量：`--count 3`
- 检查失败日志，针对性重试

### 问题：验证码接收慢

**解决方案**：
- 脚本已内置 120 秒超时
- 如果经常超时，考虑更换邮箱服务或增加间隔

## 📦 备份文件

每次修改脚本前都会自动创建备份：
```
register.py.backup.YYYYMMDD-HHMMSS
```

恢复备份：
```bash
cp register.py.backup.20260325-172250 register.py
```

## 🎯 最佳实践

1. **小批量测试**：首次使用建议 `--count 2` 测试
2. **合理间隔**：建议 3-5 秒间隔，避免触发限制
3. **使用代理**：如果在不支持的地区，务必配置代理
4. **保存结果**：JSON 文件包含所有账号信息，妥善保管
5. **错误重试**：失败的账号可以单独重试（`--count 1`）

## 📞 技术支持

如遇问题，请检查：
1. Python 版本（需要 3.7+）
2. 依赖包是否安装完整
3. 网络连接是否正常
4. 代理配置是否正确
